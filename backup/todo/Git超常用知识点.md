# Git 超常用知识点（快进来查漏补缺）

## 前言
Git是什么、有什么作用
Git是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

如果想了解Git的历史推荐阅读：[Git 的故事：这一次没这么好玩](https://mp.weixin.qq.com/s/aQlknAL6k0u1TR6hNl5mBA)，揭秘 Linus 创造 Git 背后的故事。

下面汇总了笔者日常工作中总结的 Git 最常用的知识点和用好 Git 的方法，大家可以查漏补缺并练习，将 Git 用的更好。

PS: 文章非入门教程或常用命令罗列，如果有相关诉求建议阅读官方文档或询问大模型。由于笔者水平有限，若未提及一些您认为更重要的知识点，欢迎在评论区补充~
## 知识点
### 三板斧
下面这三行命令就是 Git 的三板斧，在我工作的第一年，每天都在使用这三行命令。那时我完全不会使用别的 Git 命令，但也不妨碍我正常开发。因为当时接的需求都很简单，不会和其他人合作，只会在最后提交代码的时候使用 Git。
```
git add --all
git commit -m "commit message"
git push
```
